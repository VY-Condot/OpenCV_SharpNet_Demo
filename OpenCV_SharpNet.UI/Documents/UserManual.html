🚀 1. Getting Started

System Requirements

Component Requirement
Operating System Windows 10 / Windows 11 (64-bit)
.NET Framework .NET 8 (must be installed)
Processor Intel/AMD x64 (multi-core recommended)
Memory (RAM) 8 GB minimum (16 GB recommended)
Storage 500 MB minimum + space for images
Camera (Optional) GigE, USB3, or GenTL compatible

Installation Steps

  • Download & Install .NET 8

    Download from: https://dotnet.microsoft.com/download

  • Extract Application Files

    Unzip the application to your desired location (e.g., C:\Applications\CsplCam.Library)

  • Prepare Template Dataset

    Create folder D:\py\char_dataset and copy character templates (or use training mode to create your own)

  • Launch Application

    Double-click CsplCam.Library.exe

✅ Success: If the application window appears with "Ready" status, setup is complete!

📋 3. Basic Workflow: Loading & Processing Images

Step-by-Step: Load and Process an Image

  • Click "Browse Image" Button

    A file dialog opens. Navigate to your image folder and select one or multiple image files (PNG, JPG, TIFF supported)

    💡 Tip: You can select multiple images at once using Ctrl+Click
  • View Image in Canvas

    The first image loads in the central canvas area. Images appear in the "Selected Image" list at the top.

  • Navigate Between Images

    Use "Next Image" and "Previous Image" buttons, or click on image name in the list

  • Zoom and Pan

    • Scroll mouse wheel to zoom in/out
    • Left-click background and drag to pan around image
    • Use keyboard shortcuts for precise control

Understanding the Status Bar

Status Bar Messages:
Message Meaning
"Ready" Application idle, waiting for action
"Decode ROI in X ms" Processing complete; X milliseconds elapsed
"Image Panning mode" You're dragging/panning the canvas

"Total X images are selected" X image files loaded for batch processing

📍 4. Creating ROIs (Regions of Interest)

What is an ROI?

An ROI (Region of Interest) is a defined area on your image where you want the system to process text, barcodes, or templates. You can define multiple ROIs on a single image to recognize different areas independently.

ROI Types & Uses

ROI Type Purpose Example
Text ROI Recognize printed/typed text Product expiration date, serial number
Barcode ROI Decode 1D/2D barcodes QR code, UPC, Data Matrix
Template Match ROI Verify visual patterns Check label alignment, brand logo

Creating a Text ROI

  • Click "Add Text ROI" Button

    A new ROI box appears at the center of the image canvas

  • Position the ROI

    • Click and drag the ROI box center to move it
    • Drag the corners/edges to resize it
    • Make sure it covers the text area you want to recognize

  • Configure Settings (Right Panel)

    A settings panel appears on the right with:

    • Expected Text: Enter ground truth for validation
    • Threshold: Set confidence score minimum (0.0-1.0)
    • Min/Max Width/Height: Blob size filtering
    • Segmentation Mode: Choose preprocessing algorithm
    • Morphology: Apply Erode/Dilate filters
  • Click "Decode All ROI" to Process

    The system recognizes text in this ROI. Results appear in the table below.

Creating a Barcode ROI

  • Click "Add Barcode ROI" Button
  • Position/resize ROI to cover the barcode
  • In settings panel:
    • Barcode Format: Select specific format or "Auto" for auto-detect
    • Enable "Auto-Detect" checkbox for format flexibility
    • Enable "Run QC Check" for quality grading
  • Click "Decode All ROI" to scan and decode

ROI Visual Indicators

Understanding ROI Colors:
  • 🔲 Magenta Box: Unselected ROI
  • 🔳 Cyan Box: Currently selected ROI
  • 🔘 Yellow Squares: Resize handles (on selected ROI)
  • 🟢 Green Text: Successful character match
  • 🔴 Red Text: Failed character match

Working with Multiple ROIs

💡 Pro Tips:
  • Add multiple ROIs on the same image for parallel processing
  • Copy/paste ROI configs: Ctrl+C to copy, Ctrl+V to paste
  • Right-click ROI for context menu (copy, rename, delete, rotate)
  • All ROIs decode in parallel for fast processing
  • Save ROI layouts as JSON for batch processing

🔤 5. Text Recognition (OCR)

How OCR Works

Recognition Pipeline:
  1. 📷 Capture: Extract text region from image
  2. 🔄 Preprocess: Apply segmentation & deskewing
  3. ✂️ Segment: Find individual character bounding boxes
  4. 🔍 Recognize: Match each character against 800+ templates
  5. ✅ Validate: Compare against expected text (optional)

Segmentation Modes

Mode Best For Speed Accuracy
Balanced Most documents (default) ⚡ Fast ⭐⭐⭐⭐
High Accuracy Clean, high-quality scans ⚡ Fast ⭐⭐⭐⭐⭐
Noise Heavy Damaged/faded text 🐢 Slow ⭐⭐⭐⭐
Punctuation Symbols & punctuation ⚡⚡ Very Fast ⭐⭐⭐
Industrial Factory/manufacturing 🐢 Slow ⭐⭐⭐⭐⭐

Step-by-Step: Recognize Text in ROI

  • Create Text ROI (see Section 4)
  • Set Expected Text (optional)

    In the right panel, enter the expected text to validate results

  • Choose Segmentation Mode

    In settings, select preprocessing algorithm based on image quality

  • Adjust Blob Filters

    Set Min/Max character width and height for your font size

  • Click "Decode All ROI"

    Processing begins. Status bar shows elapsed time.

  • View Results

    • Decoded text appears in results table
    • Green boxes = matches, Red boxes = mismatches
    • Confidence scores shown per character

Understanding Recognition Confidence

Confidence Score (0.0 - 1.0):
  • 🟢 0.9 - 1.0: Excellent match, very reliable
  • 🟡 0.7 - 0.9: Good match, reliable
  • 🟠 0.5 - 0.7: Fair match, consider manual verification
  • 🔴 0.0 - 0.5: Poor match, likely incorrect

Improving Recognition Accuracy

🔧 Optimization Tips:
  • ✅ Use "High Accuracy" mode for clean documents
  • ✅ Adjust blob filters (Min/Max width/height) for font size
  • ✅ Apply morphology (Erode/Dilate) to clean noise
  • ✅ Ensure ROI tightly covers text area (no extra background)
  • ✅ Use "Train Selected ROI" to teach system new characters
  • ✅ Rotate ROI if text is tilted (right-click → Rotate)

ITF Shipping cartons

Pharmacode Medicine packaging

Step-by-Step: Decode a Barcode

  • Create Barcode ROI

    Click "Add Barcode ROI" and position box over the barcode

  • Configure Barcode Settings
    • Format: Select specific format or leave as "Auto"
    • Auto-Detect: Enable for flexibility with unknown formats
    • QC Check: Enable to get quality grades (optional)
  • Click "Decode All ROI"

    Multi-stage decoding begins:

    • Try on original grayscale
    • Try on binary (threshold)
    • Try with padding
    • Try on scaled image
    • Return result or "Failed"
  • View Decoded Value

    Decoded barcode text appears in results table

  • Check Quality Grade (if QC enabled)

    Right panel shows ISO 15415 grades (A-F scale)

Barcode Quality Grades (ISO 15415)

Grade Score Meaning Action
A 4 Excellent ✅ Accept
B 3 Good ✅ Accept
C 2 Fair ⚠️ Marginal
D 1 Poor 🔴 Investigate
F 0 Failed ❌ Reject

Quality Metrics Explained

Metric What It Measures Pass Criteria
Decode Successfully read barcode Yes = Grade A
Symbol Contrast Dark vs. light intensity ≥ 70%
Axial Non-Uniformity Shape distortion ≤ 8%
Print Growth Ink expansion/shrinkage ±15%
Modulation Pixel value variance ≥ 0.25

Troubleshooting Barcode Issues

Problem Likely Cause Solution
Decoding fails Poor barcode quality Check lighting, clean barcode, adjust ROI
Wrong format detected Auto-detect confusion Specify exact format instead of "Auto"
Low quality grade (C/D) Printing defects Improve print quality with vendor
Partial read ROI too small Expand ROI to cover entire barcode

🎓 7. Training Custom Characters

Why Train New Characters?

The system comes with ~800 pre-trained character templates. If your documents use unique fonts or characters not in the base dataset, you can train new ones!

Step-by-Step Training

  • Create a Text ROI with sample text

    Make sure it contains the characters you want to train

  • Click "Train Selected ROI" Button

    A training dialog opens showing individual character blobs

  • Review Detected Characters

    System displays each detected character blob separately

  • Enter Correct Character for Each Blob

    Type the correct character/symbol that matches each blob

  • Save Trained Characters

    Click "Save" to store templates in dataset folder

  • Use Trained Characters Immediately

    Next recognition will use newly trained templates

Training Best Practices

✅ Do's:
  • ✓ Train multiple samples of each character (different positions, sizes)
  • ✓ Use high-quality images for training
  • ✓ Train one ROI at a time
  • ✓ Verify each blob is labeled correctly
❌ Don'ts:
  • ✗ Don't mix different fonts in single training session
  • ✗ Don't train on very small or blurry characters
  • ✗ Don't mislabel characters

Where Trained Templates Are Stored

⚠️ Important: Trained templates are saved to:
D:\py\char_dataset\{character_folder}\
  • Format: Lowercase letters in "lower_a", "lower_b", etc.
  • Uppercase: "upper_A", "upper_B", etc.
  • Symbols: "slash", "comma", "colon", etc.

Delete from this folder to remove trained templates.

Using the Camera Mode

⚠️ Note: Camera integration requires compatible camera (GigE, USB3, or GenTL). Skip this section if you're only processing offline images.

Live ROI Processing

Camera mode supports saving frames in multiple formats:

🔧 10. Troubleshooting & FAQ

Common Issues & Solutions

🔴 Application Won't Start

Problem: Application crashes or won't launch
  • Solution 1: Check .NET 8 installation → dotnet --version
  • Solution 2: Check templates folder exists → D:\py\char_dataset
  • Solution 3: Clear temp files → Delete %TEMP%\CsplCam.Library
  • Solution 4: Reinstall application

🔴 Templates Not Loading

Problem: Status shows "No templates loaded"
  • Solution 1: Verify folder path in config → D:\py\char_dataset
  • Solution 2: Ensure templates exist → Files in subfolders (upper_A, lower_a, etc.)
  • Solution 3: Check file permissions → Folder readable by application
  • Solution 4: Train new templates if none available

🔴 Recognition Accuracy Low

Problem: Confidence scores are consistently low (< 0.5)
  • Solution 1: Try different Segmentation Mode (see Section 5)
  • Solution 2: Adjust Blob Filters (Min/Max width/height)
  • Solution 3: Apply Morphology (Erode to remove noise)
  • Solution 4: Rotate ROI if text is tilted
  • Solution 5: Train custom templates for your font
  • Solution 6: Improve image quality (better lighting, focus)

🔴 Barcode Decoding Fails

Problem: "Failed" status for barcode ROI
  • Solution 1: Verify barcode format (try "Auto")
  • Solution 2: Expand ROI to cover entire barcode
  • Solution 3: Check lighting (not too dark/bright)
  • Solution 4: Try different barcode if available
  • Solution 5: Manually decode from image to verify it's valid

🔴 Performance is Slow

Problem: Processing takes > 500ms per ROI
  • Solution 1: Reduce image resolution (resize before processing)
  • Solution 2: Disable unnecessary features (QC checks)
  • Solution 3: Close other applications (free up RAM)
  • Solution 4: Use "Punctuation" mode instead of "Noise Heavy"
  • Solution 5: Upgrade system hardware (more CPU cores)

🔴 Camera Won't Connect

Problem: Camera device not appearing in list
  • Solution 1: Check physical USB/GigE connection
  • Solution 2: Install camera drivers (see manufacturer docs)
  • Solution 3: Click "Enumerate Devices" button again
  • Solution 4: Restart application
  • Solution 5: Check firewall doesn't block GigE access

FAQ (Frequently Asked Questions)

Q: Can I process images in batch?

A: Yes! Load multiple images at once, define ROIs, then navigate and decode each image. Results accumulate in the table.

Q: How do I save my ROI layout?

A: Click "Save ROI" button. Configuration saves as JSON file. Later, click "Load ROI" to apply same layout to new images.

Q: Can I copy ROI settings between images?

A: Yes! Use Ctrl+C to copy an ROI, then Ctrl+V to paste. Useful for creating multiple similar ROIs.

Q: What image formats are supported?

A: PNG, JPG, JPEG, TIFF, BMP. Choose highest quality for best results.

Q: How do I improve template dataset?

A: Use "Train Selected ROI" feature. More diverse training samples = better recognition.

Q: Can I process rotated text?

A: Yes! Right-click ROI → "Rotate" to set 0°/90°/180°/270° angles. Or use auto-deskew in preprocessing.

Q: What's the difference between Barcode QC and Text Validation?

A: Barcode QC grades physical barcode quality (ISO 15415). Text Validation compares decoded text to expected value.

Q: How accurate is the recognition?

A: Typical accuracy 95-99% for high-quality images with matched templates. Accuracy depends on image quality, font, and template dataset.

Q: Can I use custom fonts?

A: Yes! Train templates for any font using "Train Selected ROI". System becomes familiar with that font's character shapes.

Q: What if I make a mistake during training?

A: Templates save with timestamps. Delete incorrect templates from dataset folder and retrain.
